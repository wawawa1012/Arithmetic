### 💡 核心数据结构与算法总结

#### 一、栈 (Stack) 的三大适用场景 (后进先出 LIFO)
1. **“洋葱模型” / 嵌套匹配：** 信息一层包着一层（如 `3[a2[c]]`、括号匹配、JSON 语法解析）。外层状态必须先**“存档 (Push)”**挂起，等待最内层逻辑处理完毕后，再**“读档 (Pop)”**向外合并。
2. **“历史清算” / 寻找下一个更大值：** 站在当前节点无法立刻得到答案，过去的元素处于“待满足”的等待状态（如“每日温度”）。单调栈允许未来某个符合条件的强力元素降临时，将历史记录像割麦子一样**批量清算出栈**。
3. **“时光倒流” / 撤销动作 (Undo)：** 只能操作“距离当前最近的一个历史动作”（如文本编辑器退格、逆波兰表达式中的连续两数计算）。

#### 二、回溯算法 (Backtracking) 的三大适用场景 (多重宇宙的探索)
在面对算法题时，一旦看到以下三个特征，**必须放弃 `for` 循环的线性思维，直接套用回溯算法的“克隆人探险”模板：**
1. **关键词：“找出所有可能的解 / 方案” (穷举组合/排列/子集)：** * **雷达：** 只要题目要求你返回 `vector<vector<int>>` 这种“包含所有结局的二维大铁桶”，且数据规模通常较小（$N \le 20$），绝对是回溯。
  * **动作：** 使用 `push_back` 拿取，派发克隆人探索，探索结束后严格执行 `pop_back` 恢复背包原状（时光倒流）。
2. **关键词：“满足某条件的 / 有效的” (限制条件下的路径探索)：**
  * **雷达：** 比如生成“有效的”括号、走迷宫、“不互相攻击”的八皇后。有规则约束着你不能瞎走。
  * **动作：** 在向下分裂平行宇宙前，先用 `if` 拦截非法分支（剪枝）。走到绝路立刻 `return` 撤销。
3. **关键词：“凑成总和 / 无限次使用” (无限火力的资源分配)：**
  * **雷达：** 给定一个目标值（Target），让你在数组里挑东西凑够它。
  * **动作：** 使用 `for (int i = start_index; ...)` 铺设传送门阵列。为了防止出现顺序不同的重复解（如 `[2,3]` 和 `[3,2]`），向下一层派发克隆人时，必须传入当前的 `i`，锁死克隆人只能向右探索，绝不回头。

## 1. 八皇后问题 (N-Queens)

**📝 题干描述：**
在一个 8x8 的国际象棋棋盘上放置 8 个皇后，要求它们互不攻击。即任意两个皇后不能处于同一行、同一列或同一斜线上。求出所有的合法摆放方案。

**💡 核心思路 (DFS + 回溯)：**
* **按行推进：** 每一行必定只有一个皇后，因此可以通过递归函数 `trace_back(int r)` 逐行向下放置，避免了复杂的同行冲突检查。
* **空间判定 (AABB思想延伸)：** 使用 `vector<point> queens` 动态记录已放置皇后的坐标。判断对角线冲突时，无需建立复杂的二维映射，直接利用数学绝对值关系：`abs(当前列 - 已存列) == abs(当前行 - 已存行)` 即可精准拦截。
* **状态回溯：** 无论下一行是否找到解，当前行的 `for` 循环在尝试下一个列位置前，必须执行 `queens.pop_back()` 弹出上一次的尝试，恢复棋盘状态（读档）。

## 2. 迷宫寻路 (Maze Pathfinding)

**📝 题干描述：**
给定一个 5x5 的二维数组表示迷宫（`0` 代表空地，`1` 代表墙壁）。要求玩家从起点 `(0, 0)` 出发，只能上、下、左、右移动，寻找一条到达终点 `(4, 4)` 的通关路径，并输出该路径的坐标序列。

**💡 核心思路 (DFS + 状态标记)：**
* **边界与障碍拦截：** 递归函数的起手式先处理异常，遇到数组越界、或者踩到墙壁 `1` 直接 `return false`。
* **防止死循环 (撒面包屑)：** 在 2D 地图上寻路，极易发生“左右横跳”的无限递归。因此在踩上安全格子后，立刻将其值修改为 `2`（标记为已访问），断绝后代递归重走回头路的可能。
* **多米诺骨牌式返回：** 使用 `if (dfs(下一步)) return true;` 进行四周探测。只要最深处摸到了终点 `(4, 4)`，这个 `true` 就会沿着调用栈一路向上传递，瞬间终止其他无意义的方向探索。
* **清理现场：** 如果四个方向全是死胡同，在 `return false` 撤退前，必须把当前格子的 `2` 恢复为 `0`，并从路径 `vector` 中弹出坐标。

## 3. 单词搜索 (Word Search)

**📝 题干描述：**
给定一个 m x n 的二维字符网格 `board` 和一个字符串单词 `target`。如果 `target` 存在于网格中，返回 `true`。单词必须按照字母顺序，通过相邻的单元格内的字母构成，其中“相邻”单元格是那些水平相邻或垂直相邻的单元格。同一个单元格内的字母在一个单词中不允许被重复使用。

**💡 核心思路 (全局搜索 + 索引追踪)：**
* **双层遍历寻找起点：** 目标单词的第一个字母可能在网格的任意位置，因此必须在外部使用双层 `for` 循环遍历整个 `board`，一旦内部的 `dfs` 命中目标，立刻跳出。
* **严格的执行顺序拦截：** 1. 检查坐标越界。
  2. 检查当前字母是否匹配 `target.at(index)`。
  3. 检查是否为最后一个字母 `index == target.size() - 1`（如果是，直接保存并宣告胜利，避免触发越界报错）。
* **完美现场保护：** 由于题目限制“同一个字母不能重复使用”，在进入下一层递归前，使用 `char temp = board[r][c]` 记住当前字母，并将棋盘上的该位置设为 `' '`（空格）作为访问标记。探索失败回溯时，再执行 `board[r][c] = temp` 将字母精准归还。

## 4. 岛屿数量 (Number of Islands)

**📝 题干描述：**
给定一个由 `'1'`（陆地）和 `'0'`（水）组成的的二维网格，计算岛屿的数量。岛屿总是被水包围，并且每座岛屿只能由水平方向或竖直方向上相邻的陆地连接形成。 (LeetCode 200)

**💡 核心思路 (DFS 推土机模式 + 性能优化)：**
* **破坏性探索 (无回溯)：** 与寻找特定路径不同，统计岛屿数量只需要将连通的陆地“消灭”即可。当发现 `'1'` 时，计数器加一，并启动 DFS 将相邻的所有 `'1'` 永久修改为 `'0'`（沉岛），因此无需在递归后执行 `恢复状态` 的回溯操作。
* **守门员拦截前置：** 在 DFS 函数的第一行进行严苛的越界与水面拦截 (`r < 0 || r >= rows` 等)，使得在 `for` 循环扩展方向时可以大胆传入新坐标，无需在循环内部做冗余判断。
* **工程级 C++ 提速：** 使用 `static const int dirs[4][2]` 存储四个扩展方向，避免递归过程中高频重复在栈上分配和销毁数组，显著降低内存与时间开销。

## 5. 全排列 (Permutations)

**📝 题干描述：**
给定一个不含重复数字的数组 `nums`，返回其所有可能的全排列。 (LeetCode 46)

**💡 核心思路 (纯粹状态回溯)：**
* **从具象空间到抽象组合：** 剥离了二维网格的上下左右，`for` 循环遍历整个 `nums` 数组本身就是克隆人在尝试各种“方向”。递归函数的层数，巧妙替代了原本无法预知深度的动态嵌套 `for` 循环。
* **备忘录机制 (Memoization)：** 引入 `vector<bool> used` 数组记录当前路径中已经选取过的数字，充当“撞墙”的拦截条件。
* **完美恢复现场 (Pop Back)：** 当克隆人完成某一条完整路径（`path.size() == nums.size()`）并返回后，必须通过 `path.pop_back()` 和 `used[i] = false` 将刚放进去的数字“拿回手里”，只有这样才能让 `for` 循环继续推进并尝试其他未选的组合分支。

## 6. 图像渲染 (Flood Fill)

**📝 题干描述：**
给定一个二维整数网格 `image` 代表一幅图像，以及一个起点 `(sr, sc)` 和一个新颜色 `color`。要求将起点及其上下左右所有相连的、且原始颜色与起点相同的像素，全部替换为新颜色。(LeetCode 733)

**💡 核心思路 (DFS 施工模式)：**
* **携带“通行证”：** 在进入 DFS 之前，必须先记录起点的初始颜色 `oldColor`。只有遇到这个颜色的色块，探路者才允许进行染色。
* **直接施工 (无需回溯)：** 染色是一个破坏性的状态覆盖过程。一旦将 `image[r][c]` 修改为新颜色，它自然就不再等于 `oldColor`，这天然起到了“撒面包屑”防止死循环的作用，因此绝对不需要 `pop_back` 或状态还原。
* **关键防呆拦截：** 如果 `oldColor == color`，必须在最外层直接拦截返回，否则会在原地触发无限递归。

## 7. 盛最多水的容器 (Container With Most Water)

**📝 题干描述：**
给定一个整数数组 `height`，数组中的每个数字代表一块垂直木板的高度。找出两块木板，使得它们与 x 轴构成的容器能装下最多的水。求最大水量。(LeetCode 11)

**💡 核心思路 (双指针 / 贪心算法降维打击)：**
* **彻底摒弃递归：** 放弃 $O(N^2)$ 的暴力双层遍历。利用物理常识：装水量取决于两板之间的“距离”和“较短板的高度”。
* **双端夹击：** 初始化两个指针 `left` 和 `right` 分别指向数组首尾，此时距离处于最大状态。
* **贪心移动策略：** 记录当前面积后，要想在距离缩短的情况下找到更大的面积，**唯一的希望就是寻找更高的板**。因此，永远只让指向“较短板”的指针向中间移动，不断更新历史最大面积，直到两指针相遇。复杂度完美降至 $O(N)$。

## 8. 对称二叉树 (Symmetric Tree)

**📝 题干描述：**
给你一个二叉树的根节点 `root` ， 检查它是否轴对称。(LeetCode 101)

**💡 核心思路 (二叉树交叉递归)：**
* **双探子交叉对比：** 判断对称不能用单个指针向下钻，必须同时派发左右两个指针（`p` 和 `q`）进行对比验证。
* **极致的防守拦截：** 递归起手直接使用 `if` 拦截所有异常情况：双节点皆空为完美对称（true）；单节点为空则结构不对称（false）；节点值不相等则镜像破裂（false）。
* **镜像递归逻辑：** 将左探子的左子树与右探子的右子树绑定验证，将左探子的右子树与右探子的左子树绑定验证。通过 `return check(p->left, q->right) && check(p->right, q->left)` 完成逻辑向上传递。

## 9. 二进制矩阵中的最短路径 (Shortest Path in Binary Matrix)

**📝 题干描述：**
给你一个 `n x n` 的二进制矩阵 `grid`，返回矩阵中最短畅通路径的长度。畅通路径是一条从左上角到右下角的路径，途经的单元格全为 `0`，且支持 8 个方向的相邻移动（平移与对角线平权）。 (LeetCode 1091)

**💡 核心思路 (BFS 水波纹扩散 / 层序遍历)：**
* **最短路径的绝对真神 (摒弃递归)：** 寻找“最短距离”时，DFS 会像无头苍蝇一样陷入盲目深搜。必须使用 `std::queue` 配合广度优先搜索 (BFS)，像水波纹一样一层层向外均匀扩散，水波第一次触碰终点时的层数，即为绝对的最短距离。
* **层级隔离机制：** 不能每取出一个点就增加步数。必须使用 `int current_layer_size = q.size()` 锁死当前层的人数，用内部 `for` 循环把当前这一圈的水滴全部处理完后，`step` 才能统一加 1。
* **入队即沉岛 (防 OOM 核心)：** 2D 网格的 BFS 极易发生“多个节点同时将同一个合法邻居塞入队列”的惨剧，导致内存爆炸。因此，在探测到合法空地并 `push` 入队后，**必须立刻**将该 `grid[nr][nc]` 修改为墙壁 `1`，实现完美的现场封锁，防止别人走回头路或重复排队。

## 10. 腐烂的橘子 (Rotting Oranges)

**📝 题干描述：**
给定一个 `m x n` 的网格，`0` 代表空格子，`1` 代表新鲜橘子，`2` 代表腐烂的橘子。每分钟，腐烂的橘子会传染上下左右相邻的新鲜橘子。返回直到没有任何新鲜橘子所需的最少分钟数。如果不可能，返回 `-1`。(LeetCode 994)

**💡 核心思路 (多源 BFS + 工程防坑)：**
* **多源点爆发：** 在第 0 分钟，可能存在多个感染源。必须在初始阶段双层遍历全图，将所有为 `2` 的节点悉数推入队列，同时统计所有为 `1` 的新鲜橘子总数（`fresh_count`）作为终局判断依据。
* **致命陷阱：动态 q.size() 坍塌：** 绝对禁止在 `for` 循环判断条件中使用 `i < q.size()`。因为在传染过程中，新被咬的橘子会不断入队，导致队伍长度动态增加，从而造成不同分钟的时间线混杂。必须使用 `int current_layer_size = q.size()` 提前锁死当前分钟的行动名额。
* **物理世界的并行逻辑：** 即使这一分钟内 `fresh_count` 降为了 `0`，也不能急于在内层循环立刻 `return time`。因为这 1 分钟在物理宇宙中尚未结束，必须让这一层的 `for` 循环跑完，交给外层统一进行 `time++`，才能保证时间的准确无误。

## 11. 长度最小的子数组 (Minimum Size Subarray Sum)

**📝 题干描述：**
给定一个含有 `n` 个正整数的数组和一个正整数 `target`。找出该数组中满足其和 $\ge target$ 的长度最短的连续子数组，并返回其长度。如果不存在符合条件的子数组，返回 0。 (LeetCode 209)

**💡 核心思路 (滑动窗口的降维打击)：**
* **使用场景与核心优势：** 当题目要求在数组/字符串中寻找满足特定条件的**连续子区间**（如“最长”、“最短”、“总和等于”），且暴力解法需要 $O(N^2)$ 的双层循环时，直接套用滑动窗口。通过左右两个同向移动的指针（绝不走回头路），将时间复杂度极致压缩为 $O(N)$。
* **状态累加与被动收缩：** 维护一个变量 `sum` 记录当前窗口状态。右指针主动探测边界并累加 `sum += nums[right]`。当 `sum >= target` 时，说明当前窗口已达标。为了探寻“最短”极限，必须强迫左指针向右逼近（`sum -= nums[left]; left++;`）吐出冗余元素，直到窗口再次违规，右指针才继续前行。
* **工程兜底 (INT_MAX 陷阱)：** 寻找最小长度时，初始标记必须设为一个极大值（如 `INT_MAX`）。在算法终点 `return` 时，必须使用三目运算符拦截：若 `min_len` 未被更新，说明全数组之和仍不及 `target`，严格遵照题意兜底返回 `0`。

## 12. 每日温度 (Daily Temperatures)

**📝 题干描述：**
给定一个整数数组 `temperatures` ，表示每天的温度。返回一个数组 `answer` ，其中 `answer[i]` 是指对于第 `i` 天，下一个更高温度出现在几天后。如果气温在这之后都不会升高，请在该位置用 `0` 来代替。 (LeetCode 739)

**💡 核心思路 (单调栈：站在未来，清算过去)：**
* **物理降维 ($O(N^2)$ 到 $O(N)$)：** 放弃主动向未来寻找的暴力双层循环。改用 `std::stack` 维护一个“历史悬案室”，让未来的每一天主动来清算过去的等待。每个元素最多只会入栈和出栈各一次，达到极致的 $O(N)$ 效率。
* **存储索引而非具体值：** 栈内绝对不能存储具体的温度数值，而是必须存储**日期的索引 (`i`)**。因为题目要求的是“等待的天数”，只有保留索引，才能在出栈时通过 `今天索引 - 栈顶索引` 算出精确的天数差，并精准填入 `answer[d]`。
* **判官与苦主 (入栈与出栈逻辑)：** 1. 遍历到第 `i` 天时，今天的温度化身“判官”。
  2. 只要栈不为空，且今天温度**大于**栈顶索引对应的历史温度，说明栈顶的日子终于等到了升温！将其弹出（结案），并计算天数填入答案数组。循环此操作直到踢不开门（遇到比今天更热的历史记录）。
  3. 历史清算完毕后，今天自己作为新的“苦主”入栈排队，等待未来的某一天被更热的日子解救。栈内自底向上永远保持温度递减的状态（单调递减栈）。

## 13. 逆波兰表达式求值 (Evaluate Reverse Polish Notation)

**📝 题干描述：**
给你一个字符串数组 `tokens` ，表示一个根据“逆波兰表示法”（后缀表达式）表示的算术表达式。请你计算该表达式并返回结果。有效的算符为 `+`、`-`、`*` 和 `/`。(LeetCode 150)

**💡 核心思路 (栈的后进先出 LIFO)：**
* **编译器的底层引擎：** 逆波兰表达式本质上是二叉语法树的“后序遍历（左右根）”结果。由于它去除了括号的优先级干扰，非常适合使用 `std::stack` 进行线性求值。
* **数字入栈，符号清算：** 遍历 `tokens`。如果遇到数字字符串，通过 `stoi(token)` 转化为整型并压入栈中。如果遇到运算符，则说明需要进行一次局部计算。
* **致命陷阱 (左右互搏)：** 由于栈是后进先出，遇到运算符时，**第一个** `pop` 出来的必定是**右操作数**，**第二个** `pop` 出来的才是**左操作数**。严格遵守 `左操作数 [运算符] 右操作数` 计算后，必须将局部结果重新 `push` 回栈中，供后续计算使用。
* **C++ 工程技巧 (Switch 优化)：** C++ 的 `switch` 不支持 `string`，但由于运算符字符串长度必定为 1，可通过 `switch(token[0])` 提取 `char` 字符进行优雅的分支匹配。

## 14. 字符串解码 (Decode String)

**📝 题干描述：**
给定一个经过编码的字符串，返回它解码后的字符串。编码规则为: `k[encoded_string]`，表示其中方括号内部的 `encoded_string` 正好重复 `k` 次。保证 `k` 是正整数。(LeetCode 394)

**💡 核心思路 (双栈架构 / 游戏存档读档理论)：**
* **跨越时空的状态保存：** 面对未知的多层嵌套（如 `3[a2[c]]`），不能仅仅使用局部变量。必须在外部维护两个“永久背包”：`currNum`（当前倍数）和 `currSTR`（当前收集的字符串）。
* **进入传送门 (`[` 入栈存档)：** 当遇到 `[` 时，意味着即将进入一个全新的深层副本。此时必须将旧世界打到的怪（`currSTR`）和将要刷的次数（`currNum`）**推入各自的栈中进行物理存档**。存档后，将背包清零（`currSTR = ""`, `currNum = 0`），轻装进入副本。
* **打穿副本 (`]` 出栈读档)：** 当遇到 `]` 时，深层探索结束。此时手中拿着副本的产出（当前的 `currSTR`）。从栈中弹出历史倍数进行翻倍复制，再从栈中弹出历史旧存档，将复制好的新产出拼接在旧存档的末尾，完成两个世界的融合。
* **黄金数字累加：** 处理多位字符串数字时，统一使用公式 `currNum = currNum * 10 + (c - '0')`，优雅且安全。

## 15. 反转链表 (Reverse Linked List)

**📝 题干描述：**
给你单链表的头节点 `head` ，请你反转链表，并返回反转后的链表。(LeetCode 206)

**💡 核心思路 (双指针 / 链表物理防断裂)：**
* **链表的物理特性：** 链表节点在内存中是不连续分布的，完全依靠 `next` 指针维持联系。在修改指针朝向（反转）时，一旦提前断开连接，后续链表将瞬间造成内存丢失（内存泄露）。
* **三步走安全反转法：** 需要维护两个指针：`prev`（初始为 `nullptr`）和 `curr`（初始为 `head`）。
  1. **探路备份：** 在 `curr` 断开与下一节点的连接前，必须先使用临时指针截获其后继节点 `ListNode* temp = curr->next;`。
  2. **原地转身：** 安全备份后，断开当前节点并反转朝向：`curr->next = prev;`。
  3. **双指针推进：** `prev` 移动到当前位置 (`prev = curr;`)，`curr` 移动到刚才备份的位置 (`curr = temp;`)。
* **C++ 内存防坑 (野指针与段错误)：** 声明指针后必须立即初始化。绝对禁止在未分配实际对象内存空间的情况下，声明 `ListNode* temp;` 并直接对其成员进行赋值（如 `temp->next = ...`），这会触发严重的 `Segmentation Fault` (段错误)。指针本身即代表地址，直接使用 `temp = curr->next` 即可完成保存。

## 16. 括号生成 (Generate Parentheses)

**📝 题干描述：**
数字 `n` 代表生成括号的对数，请你设计一个函数，用于能够生成所有可能的并且有效的括号组合。(LeetCode 22)

**💡 核心思路 (回溯算法 / 平行宇宙的分裂与坍缩)：**
* **宇宙生存法则：** 递归回溯的本质是探索所有可能性，但受法则约束。
  1. **左分支限制：** 只要已使用的左括号数量 `< n`，就可以肆无忌惮地探索“加左括号”的宇宙。
  2. **右分支限制：** 只有在当前字符串中，右括号数量**严格小于**左括号数量时，才能探索“加右括号”的宇宙（防止出现 `)(` 等非法结构）。这两个法则平级独立，绝非嵌套关系。
* **传值 vs 传引用 (回溯工程手感)：**
  * **不弄脏当前时空 (传值)：** 在向下层递归传参时，绝不能使用 `currentStr += "("` 修改当前变量自身。必须使用 `currentStr + "("` 生成一个全新的克隆体传入下一层，确保当前宇宙的时空在探索归来后依然纯洁（时光倒流）。
  * **共享最终结局 (传引用)：** 收集结局的大数组必须定义为 `vector<string>& res`（带有引用符 `&`），相当于在村口放置一个共享大铁桶，确保所有平行宇宙的克隆人在死亡（`return`）前，能将合法答案汇聚到同一物理内存中。

## 17. 子集 (Subsets)

**📝 题干描述：**
给你一个整数数组 `nums` ，数组中的元素互不相同。返回该数组所有可能的子集（幂集）。解集不能包含重复的子集。你可以按任意顺序返回解集。(LeetCode 78)

**💡 核心思路 (二叉选择回溯 / 共享背包理论)：**
* **拿与不拿的平行宇宙：** 对于数组中的每一个数字，闯关者都面临两个极其纯粹的选择：拿进背包，或者无视跳过。
* **正统回溯手法 (时光倒流的橡皮擦)：** 为了防止内存爆炸，全宇宙必须共享同一个实体背包 `vector<int>& path`。在探索“拿”的分支时，执行 `path.push_back()` 并向下层递归；当该层递归结束（死回来）时，必须**亲手执行 `path.pop_back()` 将刚放进去的数字掏出来**，使背包恢复一尘不染的空状态，随后才能干干净净地去探索“不拿”的分支。
* **绝对对称性与防爆核弹：** 递归的核心在于操作的绝对对称，有 `push` 必有对应的 `pop`。因此，背包会在探索完毕后自动归零，**绝对禁止**在递归内部手动调用 `path.clear()`，否则会瞬间清空其他平行宇宙的历代状态，导致程序崩溃。

## 18. 组合总和 (Combination Sum)

**📝 题干描述：**
给你一个无重复元素的整数数组 `candidates` 和一个目标整数 `target`。找出 `candidates` 中可以使数字和为目标数 `target` 的所有不同组合。`candidates` 中的同一个数字可以无限制重复被选取。(LeetCode 39)

**💡 核心思路 (无限火力回溯 / For 循环传送门)：**
* **防去重的核心铁律 (start_index)：** 题目要求 `[2, 3]` 和 `[3, 2]` 视为同一种组合。为了封死“回头路”，必须引入 `start_index` 机制。使用 `for (int i = start_index; i < candidates.size(); i++)` 铺设传送门。
* **无限火力的下潜传参：** 由于当前武器可以被无限次使用，向下一层递归派发克隆人时，传入的起始点**必须是当前的循环变量 `i`**（而不是 `i + 1`，也不是万年不变的 `start_index`）。这确保了克隆人可以继续使用当前武器，或选择更右侧的武器，但永远无法回头拿取左侧的武器。
* **优雅的状态传递 (不弄脏时空)：** 对于基本数据类型的状态（如 Boss 剩余血量 `target`），无需在老父亲的时空里执行 `target -= 伤害` 然后再回溯加血。直接在向下递归的传参括号内执行 `target - candidates[i]`。这样系统会复印一个扣除血量后的新数字传递给下一层，老父亲时空中的 `target` 本体始终保持纯洁。

## 19. 环形链表 (Linked List Cycle)

**📝 题干描述：**
给你一个链表的头节点 `head` ，判断链表中是否有环。如果链表中有某个节点，可以通过连续跟踪 `next` 指针再次到达，则链表中存在环。返回 `true` 或 `false`。(LeetCode 141)



**💡 核心思路 (快慢指针 / 龟兔赛跑)：**
* **物理降维 ($O(1)$ 空间判环)：** 放弃使用哈希表记录已访问节点的 $O(N)$ 空间暴力做法。利用物理学的相对运动原理，引入速度差。
* **双指针追击：** 设置慢指针 `slow`（每次 1 步）和快指针 `fast`（每次 2 步）。如果链表无环，`fast` 必定先撞到尽头 (`nullptr`)。如果链表有环，`fast` 和 `slow` 必将在环内进行无限追逐。由于相对速度差恒定为 1，`fast` 最终必然会以每次逼近 1 步的精确度，从后方套圈并与 `slow` 重合 (`fast == slow`)。
* **致命陷阱 (短路求值与段错误)：** 在 `while` 循环条件中，必须严格防范对空指针解引用。由于 `fast` 每次需要跳跃两步 (`fast->next->next`)，循环条件必须严格按顺序写为 `fast != nullptr && fast->next != nullptr`。利用 C++ 的短路求值特性，当 `fast` 为空时直接阻断后续判定，防止 `Segmentation Fault` 惨案。

## 20. 两数之和 (Two Sum)

**📝 题干描述：**
给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出和为目标值 `target` 的那两个整数，并返回它们的数组下标。假设每种输入只会对应一个答案。(LeetCode 1)



**💡 核心思路 (哈希表 / 时间换空间降维)：**
* **打破线性未来 ($O(N^2)$ 到 $O(N)$)：** 放弃双层 `for` 循环暴力向后匹配的做法。改为边向后遍历，边将当前的自己存入历史记录，并向“过去”寻找匹配项。
* **历史账本讨债：** 维护一个 `unordered_map<int, int>` 作为历史账本（键为 `数字`，值为 `索引下标`）。遍历到 `nums[i]` 时，计算出达成目标所需的“债主” (`debt = target - nums[i]`)。
* **瞬间检索：** 利用哈希表 $O(1)$ 的检索特性，使用 `mp.contains(debt)` 瞬间判断历史账本中是否存在所需数字。若存在，直接返回 `{mp[debt], i}`。若不存在，将当前数字及下标 `mp[nums[i]] = i` 登记入册，供未来的数字讨债。
* **工程兜底：** 即使题目保证必定有解，在 C++ 函数末尾仍需加上 `return {};` 以应对编译器警告，保证代码的工程严谨性。

## 21. 电话号码的字母组合 (Letter Combinations of a Phone Number)

**📝 题干描述：**
给定一个仅包含数字 `2-9` 的字符串，返回所有它能表示的字母组合。答案可以按任意顺序返回。数字到字母的映射与老式电话按键相同。(LeetCode 17)

**💡 核心思路 (多叉树回溯 / 传值复印机模型)：**
* **全局密码本映射：** 建立全局静态数组 `const string letterMap[10]` 统一管理数字到字符串的映射逻辑。
* **单一职责原则：** 递归函数的职责是且仅是处理**当前单一深度的按键**。通过 `for (char c : letters)` 横向展开当前按键的所有可能字母，然后派发克隆人向下递归处理 `index + 1` 的下一个按键。绝对禁止在递归内部嵌套 `while` 循环试图包揽后续按键，否则会引发无限递归灾难。
* **传值复印与时光倒流：** 面对体积较小、具备短字符串优化 (SSO) 的 `string` 类型状态 `path`，舍弃繁琐的 `&` 引用与 `push/pop` 组合。直接在递归传参处使用 `path + c`，利用底层隐式拷贝生成全新的复印件传入下一层。深层探险的变动被完美隔离，老父亲的时空纯洁无瑕。
* **防呆兜底：** 若初始输入为空字符串 `""`，需在主函数入口直接拦截并返回空集，防止回溯引擎将其误判并存入一个空字符串 `[""]` 导致用例失败。
