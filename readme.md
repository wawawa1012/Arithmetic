## 1. 八皇后问题 (N-Queens)

**📝 题干描述：**
在一个 8x8 的国际象棋棋盘上放置 8 个皇后，要求它们互不攻击。即任意两个皇后不能处于同一行、同一列或同一斜线上。求出所有的合法摆放方案。

**💡 核心思路 (DFS + 回溯)：**
* **按行推进：** 每一行必定只有一个皇后，因此可以通过递归函数 `trace_back(int r)` 逐行向下放置，避免了复杂的同行冲突检查。
* **空间判定 (AABB思想延伸)：** 使用 `vector<point> queens` 动态记录已放置皇后的坐标。判断对角线冲突时，无需建立复杂的二维映射，直接利用数学绝对值关系：`abs(当前列 - 已存列) == abs(当前行 - 已存行)` 即可精准拦截。
* **状态回溯：** 无论下一行是否找到解，当前行的 `for` 循环在尝试下一个列位置前，必须执行 `queens.pop_back()` 弹出上一次的尝试，恢复棋盘状态（读档）。

## 2. 迷宫寻路 (Maze Pathfinding)

**📝 题干描述：**
给定一个 5x5 的二维数组表示迷宫（`0` 代表空地，`1` 代表墙壁）。要求玩家从起点 `(0, 0)` 出发，只能上、下、左、右移动，寻找一条到达终点 `(4, 4)` 的通关路径，并输出该路径的坐标序列。

**💡 核心思路 (DFS + 状态标记)：**
* **边界与障碍拦截：** 递归函数的起手式先处理异常，遇到数组越界、或者踩到墙壁 `1` 直接 `return false`。
* **防止死循环 (撒面包屑)：** 在 2D 地图上寻路，极易发生“左右横跳”的无限递归。因此在踩上安全格子后，立刻将其值修改为 `2`（标记为已访问），断绝后代递归重走回头路的可能。
* **多米诺骨牌式返回：** 使用 `if (dfs(下一步)) return true;` 进行四周探测。只要最深处摸到了终点 `(4, 4)`，这个 `true` 就会沿着调用栈一路向上传递，瞬间终止其他无意义的方向探索。
* **清理现场：** 如果四个方向全是死胡同，在 `return false` 撤退前，必须把当前格子的 `2` 恢复为 `0`，并从路径 `vector` 中弹出坐标。

## 3. 单词搜索 (Word Search)

**📝 题干描述：**
给定一个 m x n 的二维字符网格 `board` 和一个字符串单词 `target`。如果 `target` 存在于网格中，返回 `true`。单词必须按照字母顺序，通过相邻的单元格内的字母构成，其中“相邻”单元格是那些水平相邻或垂直相邻的单元格。同一个单元格内的字母在一个单词中不允许被重复使用。

**💡 核心思路 (全局搜索 + 索引追踪)：**
* **双层遍历寻找起点：** 目标单词的第一个字母可能在网格的任意位置，因此必须在外部使用双层 `for` 循环遍历整个 `board`，一旦内部的 `dfs` 命中目标，立刻跳出。
* **严格的执行顺序拦截：** 1. 检查坐标越界。
  2. 检查当前字母是否匹配 `target.at(index)`。
  3. 检查是否为最后一个字母 `index == target.size() - 1`（如果是，直接保存并宣告胜利，避免触发越界报错）。
* **完美现场保护：** 由于题目限制“同一个字母不能重复使用”，在进入下一层递归前，使用 `char temp = board[r][c]` 记住当前字母，并将棋盘上的该位置设为 `' '`（空格）作为访问标记。探索失败回溯时，再执行 `board[r][c] = temp` 将字母精准归还。

## 4. 岛屿数量 (Number of Islands)

**📝 题干描述：**
给定一个由 `'1'`（陆地）和 `'0'`（水）组成的的二维网格，计算岛屿的数量。岛屿总是被水包围，并且每座岛屿只能由水平方向或竖直方向上相邻的陆地连接形成。 (LeetCode 200)

**💡 核心思路 (DFS 推土机模式 + 性能优化)：**
* **破坏性探索 (无回溯)：** 与寻找特定路径不同，统计岛屿数量只需要将连通的陆地“消灭”即可。当发现 `'1'` 时，计数器加一，并启动 DFS 将相邻的所有 `'1'` 永久修改为 `'0'`（沉岛），因此无需在递归后执行 `恢复状态` 的回溯操作。
* **守门员拦截前置：** 在 DFS 函数的第一行进行严苛的越界与水面拦截 (`r < 0 || r >= rows` 等)，使得在 `for` 循环扩展方向时可以大胆传入新坐标，无需在循环内部做冗余判断。
* **工程级 C++ 提速：** 使用 `static const int dirs[4][2]` 存储四个扩展方向，避免递归过程中高频重复在栈上分配和销毁数组，显著降低内存与时间开销。

## 5. 全排列 (Permutations)

**📝 题干描述：**
给定一个不含重复数字的数组 `nums`，返回其所有可能的全排列。 (LeetCode 46)

**💡 核心思路 (纯粹状态回溯)：**
* **从具象空间到抽象组合：** 剥离了二维网格的上下左右，`for` 循环遍历整个 `nums` 数组本身就是克隆人在尝试各种“方向”。递归函数的层数，巧妙替代了原本无法预知深度的动态嵌套 `for` 循环。
* **备忘录机制 (Memoization)：** 引入 `vector<bool> used` 数组记录当前路径中已经选取过的数字，充当“撞墙”的拦截条件。
* **完美恢复现场 (Pop Back)：** 当克隆人完成某一条完整路径（`path.size() == nums.size()`）并返回后，必须通过 `path.pop_back()` 和 `used[i] = false` 将刚放进去的数字“拿回手里”，只有这样才能让 `for` 循环继续推进并尝试其他未选的组合分支。

## 6. 图像渲染 (Flood Fill)

**📝 题干描述：**
给定一个二维整数网格 `image` 代表一幅图像，以及一个起点 `(sr, sc)` 和一个新颜色 `color`。要求将起点及其上下左右所有相连的、且原始颜色与起点相同的像素，全部替换为新颜色。(LeetCode 733)

**💡 核心思路 (DFS 施工模式)：**
* **携带“通行证”：** 在进入 DFS 之前，必须先记录起点的初始颜色 `oldColor`。只有遇到这个颜色的色块，探路者才允许进行染色。
* **直接施工 (无需回溯)：** 染色是一个破坏性的状态覆盖过程。一旦将 `image[r][c]` 修改为新颜色，它自然就不再等于 `oldColor`，这天然起到了“撒面包屑”防止死循环的作用，因此绝对不需要 `pop_back` 或状态还原。
* **关键防呆拦截：** 如果 `oldColor == color`，必须在最外层直接拦截返回，否则会在原地触发无限递归。

## 7. 盛最多水的容器 (Container With Most Water)

**📝 题干描述：**
给定一个整数数组 `height`，数组中的每个数字代表一块垂直木板的高度。找出两块木板，使得它们与 x 轴构成的容器能装下最多的水。求最大水量。(LeetCode 11)

**💡 核心思路 (双指针 / 贪心算法降维打击)：**
* **彻底摒弃递归：** 放弃 O(N^2) 的暴力双层遍历。利用物理常识：装水量取决于两板之间的“距离”和“较短板的高度”。
* **双端夹击：** 初始化两个指针 `left` 和 `right` 分别指向数组首尾，此时距离处于最大状态。
* **贪心移动策略：** 记录当前面积后，要想在距离缩短的情况下找到更大的面积，**唯一的希望就是寻找更高的板**。因此，永远只让指向“较短板”的指针向中间移动，不断更新历史最大面积，直到两指针相遇。复杂度完美降至 O(N)。

## 8. 对称二叉树 (Symmetric Tree)

**📝 题干描述：**
给你一个二叉树的根节点 `root` ， 检查它是否轴对称。(LeetCode 101)

**💡 核心思路 (二叉树交叉递归)：**
* **双探子交叉对比：** 判断对称不能用单个指针向下钻，必须同时派发左右两个指针（`p` 和 `q`）进行对比验证。
* **极致的防守拦截：** 递归起手直接使用 `if` 拦截所有异常情况：双节点皆空为完美对称（true）；单节点为空则结构不对称（false）；节点值不相等则镜像破裂（false）。
* **镜像递归逻辑：** 将左探子的左子树与右探子的右子树绑定验证，将左探子的右子树与右探子的左子树绑定验证。通过 `return check(p->left, q->right) && check(p->right, q->left)` 完成逻辑向上传递。
